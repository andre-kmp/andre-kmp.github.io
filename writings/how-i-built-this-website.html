How I built this website

July 2026.

Introduction

The previous version of my website was very beautiful, and I was genuinaly proud of it. I built it in a few weeks, starting from choosing a reference to inspire me, color palette and a typography. Then I added tons of AI generated code (no shame) to make it real.

However, after it was ready, I couldn't publish anything I planned. Although I was proud, I was also feeling the whole thing didn't belong me. Everytime I thought about touching the code again, I felt I didn't know where to start.

Feelings are keys for needs. Many months later, started following, with no expectations, the getting started modules of MDN. What a surprise when I understood, semantically, how to write and read HTML and CSS code in a whole!

I came back to my website directory, deleted the html and css files and re-started from scratch.

The process can be divided in five interchangeable steps: conception, basic structuring of html, css and js files, content production, and responsibility.

Conception

Here are the requisites I first listed:

  • A website to promote and document my projects and ideas
  • Needs to be durable and perpetual
  • I also want to introduce myself on it
  • I want it to be navigable on both desktop and mobile

Then I sketched a few home-page and content-page ideas and explored color palettes possibilities from Color Hunt.

I coded the ideas to see them in the screen. This is was when I started choosing typography at Google Fonts.

Basic structuring of html, css and js files

A few concepts are fundamentally important during the work, and I consider that starting learning them properly made all the difference:

  1. Margin, padding and border
  2. Selector types (classes, elements, ids), pseudo-classes and pseudo-elements
  3. Emphasis, importance and associated semantics

There is no need to include a description of the structures because the code is available at repository for anyone who wants to read it.

Content production

Here am I. The purpose of it all. Nothing much to describe: writing in a text editor, adding html tags, rendering, re-reading, (a loop!).

Responsivity

This is the last thing I want to work on. I want all pages completed to decide how I want them to behave in thicker screens.

Next steps

I am very happy with the results, but there is a few things I still want to do:

  • Add a language switcher (goal: pt_BR, my native language)
  • Include a light-theme mode

And this is all by now. I am neither a designer or a front-end developer, but I find web-designing delightful and enjoy related topics like color theory. If you'd like to ask anything or have a conversation about this topic, please feel free to e-mail me at andreklaic@gmail.com.
